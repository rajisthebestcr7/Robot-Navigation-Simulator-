## Repository Projects

### hello_robot.py

My first Python program and the starting point of my programming journey.

**Challenge & Lesson Learned**

Since this was my first Python project, one of the biggest challenges was understanding how a Python program is structured and how code executes line by line. Although the project was simple, it helped me gain confidence in writing, running, and modifying code on my own.



### robot_move.py

A simple robot movement simulator that introduces variables, position tracking, and basic robotics concepts.

**Challenge & Lesson Learned**

One challenge was keeping track of the robot's position accurately after each movement. Through testing and small corrections, I learned how variables can represent a robot's location and how small logic mistakes can affect the final outcome of a program.



### calculator.py

A calculator program designed to strengthen my understanding of user input, calculations, and decision-making.

**Challenge & Lesson Learned**

While building the calculator, I needed to ensure that user input was handled correctly and that calculations produced the expected results. This project improved my understanding of conditional statements, program flow, and the importance of testing different user inputs.



### guessing_game.py

An interactive number guessing game that helped me practice loops, conditions, and user interaction.

**Challenge & Lesson Learned**

One challenge was creating a loop that continued running until the correct answer was guessed. Testing different scenarios helped me better understand how loops and conditions work together to create interactive programs.



### navigation_challenge.py

The largest project in this repository. This robotics-inspired navigation simulator evolved through multiple versions and became my most detailed programming project so far.

**Challenge & Lesson Learned**

This project presented the most challenges because it evolved through multiple versions. Throughout development, I worked through syntax errors, debugging issues, navigation logic problems, and file management mistakes. In one instance, I accidentally placed code into the wrong file and had to use GitHub to restore and organize my work correctly. These experiences taught me the importance of testing, debugging, version control, and making gradual improvements as a project grows.

## Development Journey

The Robot Navigation Simulator was developed through multiple versions, with each update adding new features and improving the overall experience.

### Version 1

Created the initial robot navigation simulator.

### Version 2

Added a goal position and mission completion system.

### Version 3

Implemented obstacle detection to prevent the robot from moving into blocked spaces.

### Version 4

Added move tracking and an efficiency scoring system.

### Version 5

Introduced a visual grid display and boundary detection to improve navigation feedback.

### Version 6

Added manual and automatic navigation modes, allowing the robot to be controlled by the user or navigate independently.

### Version 7

Expanded the simulator with multiple maps and difficulty levels to create different navigation challenges.

### Version 8

Improved the robot's automatic navigation and pathfinding logic to make movement decisions more effectively.

### Version 9

Added a navigation statistics dashboard to summarize mission performance and efficiency.



## Challenges I Faced

During development, I encountered several challenges that helped me improve my programming and problem-solving skills.

* Some versions introduced syntax errors that prevented the program from running correctly. Learning how to identify and resolve these issues improved my debugging skills.
* While working on later versions of the project, I accidentally placed code into the wrong file within the repository. Correcting this mistake taught me the importance of file organization and version control.
* Several navigation features did not work as expected during initial testing. I spent time reviewing the logic, making corrections, and testing different scenarios until the simulator behaved correctly.
* As the project grew, managing multiple updates became more challenging. This helped me understand how software projects evolve through continuous improvement and version tracking.
* Learning GitHub was challenging at first because I needed to understand commits, version history, and how to recover from mistakes. Over time, I became much more comfortable using GitHub to manage projects.

These experiences taught me that mistakes are a normal part of programming and that persistence is often just as important as technical knowledge.
